## 一、摘要
EM-Core V1.1 是一套**端侧原生**的通用 AI Agent 基础架构。整体实现认知、记忆、执行三层彻底解耦，搭建全链路时序闭环体系，支持终身自成长、跨设备迁移、标准化量产落地。

行业主流 Agent 多采用「Prompt 串联+向量检索」的拼凑式方案，普遍存在无状态、无长效记忆、行为不稳定、安全不可控等结构性缺陷。

EM-Core 系列三大体系同源共生、各擅其场。本 Agent 版本**依托 EM-Core-HR 人形机器人原生智能内核保留核心认知记忆逻辑，复用 EM-Core-AD 自动驾驶体系成熟的双漏斗隔离机制**。针对纯数字智能体场景，对物理硬件、车载专属领域模块做**场景适配性轻量化精简**，打造轻量、干净、可收敛、可长期迭代的端侧数字智能底座。

V1.1 为**架构冻结终稿**。全局模块编号、总线协议、安全体系、并发模型、记忆规则、会话时序全部锁定。标志着 AI Agent 从脚本式零散智能，正式迈入结构化、体系化、工业化的原生智能阶段。

## 二、体系血缘与顶层设计思想
### 2.1 架构溯源
EM-Core 全体系同源递进、分层衍生，三套产品形态**逻辑自洽、一脉相承、场景互补**，无优劣之分，仅应用领域不同：

1. **本源根底座：EM-Core-HR 人形机器人**
整套生态的智能根基。拥有完整认知、记忆、执行、物理躯体适配能力，具备最高等级的人格自收敛能力，面向物理机器人全域智能场景。

2. **衍生分支一：EM-Core-AD 自动驾驶体系**
继承人形核心认知与记忆逻辑，面向车载安全场景专属迭代。新增车辆安全体系、驾驶员适配机制，**原创落地双漏斗记忆隔离架构**，专为行车安全、人车交互场景深度优化。

3. **衍生分支二：EM-Core-Agent 数字智能体**
回归人形原生认知、记忆核心逻辑，**完整复用 AD 体系验证成熟的双漏斗隔离核心机制**。针对纯数字场景，对车载专属机制、物理躯体适配、多临时用户等**非数字领域模块做轻量化裁剪**，专注服务桌面与移动端数字自动化、数字交互智能场景。

**最终架构形态总结**：
**内核继承人形、隔离机制复用自动驾驶、形态适配纯数字**，是同源架构在数字智能领域的专属轻量化落地形态。

### 2.2 V1.1 设计定调
- 底层智能逻辑：100% 继承人形机器人原生认知规则
- 核心架构特色：复用成熟双漏斗双层记忆隔离机制
- 产品形态定位：纯数字、无硬件负担、端侧轻量化通用数字智能体

## 三、整体架构总览
### 3.1 三层严格解耦架构
整体架构永久锁定单向层级关系：
**ECC 认知层（决策）→ MLNF 记忆层（经验）→ MCC 执行层（落地）**

| 层级 | 核心职责 | 权限禁止项 |
|------|----------|------------|
| ECC 认知大脑 | 唯一意图理解、唯一安全仲裁、唯一全局调度 | 无任何底层系统执行权限 |
| MLNF 记忆中枢 | 沉淀用户偏好、任务经验、核心技能与智能人格 | 不参与推理、不判定意图、不修改决策 |
| MCC 执行小脑 | 纯状态机驱动，落地所有原子操作与自动化任务 | 无 LLM 推理、无自主决策、无方案修改能力 |

三层权责绝对隔离、单向依赖、双向总线通信。彻底杜绝跨层乱调用、逻辑耦合、智能黑盒等行业通病。

### 3.2 双总线全局通信体系
全系统仅保留两条标准通信总线，所有模块交互必须通过总线转发，禁止跨模块直连。保障系统可审计、可热更新、可模块化替换。

| 总线名称 | 通信对象 | 核心传输内容 |
|---------|----------|--------------|
| MemoryBus | 认知层 ↔ 记忆层 | 记忆检索、经验写入、I值同步、会话锁管控 |
| CerebellumBus | 认知层 ↔ 执行层 | 任务下发、调度指令、故障回执、状态同步、熔断信号 |

### 3.3 全局统一并发模型
系统采用双维度并发架构，兼顾多会话并行与单任务高效执行，全程数据隔离、时序可控。

- **多会话维度**：不同会话的队列、缓存、记忆锁、沙箱完全独立，互不干扰、互不污染。
- **同会话维度**：基于 DAG 依赖拓扑，无依赖任务多线程并行执行，强依赖任务自动串行排队。

搭配全局资源锁、会话只读锁、分支故障隔离、动态负载降级机制，实现高并发不乱、多任务不崩、异常分支不污染长效记忆。

### 3.4 固定知识底座
系统内置独立只读知识底座，包含数字世界操作规则与全局安全策略库。整体存储体积＜300KB，轻量化无负载。

该底座为系统底层基准规则，**不参与记忆衰减迭代、不参与经验层级晋升、不允许运行时动态写入**。全程固定不变，杜绝智能漂移、规则错乱、安全底线松动问题。

## 四、三大中枢核心能力
### 4.1 ECC 认知大脑（12模块 · 唯一决策源）
ECC 是整套 EM-Core-Agent 数字智能体的唯一入口、唯一智能上限、唯一安全仲裁中心。承载情境解析、意图拆解、隐私检测、风险规则匹配、伦理签名、记忆交互、会话缓存、元认知自省、状态机管控、前端反馈聚合全链路能力。

内置云端/本地双 LLM 动态调度能力，支持离线兜底运行。通过 ECC-12 双向网关统一管控并发调度、熔断转发、跨设备会话路由。全系统所有智能收敛于 ECC，所有风险拦截于 ECC，所有长效记忆准入权限受控于 ECC。

### 4.2 MLNF-Mem 双漏斗记忆中枢（60模块 · 自成长内核）
MLNF 是智能体人格、交互默契、任务技能、长期智能稳定性的唯一来源，采用**经过AD体系验证成熟、本体系完整沿用的原创双漏斗架构**，分工明确、机制互补。

| 维度 | 漏斗一：用户画像 | 漏斗二：Agent自成长 |
|------|------------------|---------------------|
| 核心目的 | 懂你——刻画用户工作习惯与交互偏好 | 练己——沉淀智能体任务经验与执行策略 |
| 分类逻辑 | 按用户领域划分（工作/创作/社交/生活/学习/通用） | 按任务类型划分（文件/代码/检索/协作/数据/自动化/通用） |
| 记忆层级 | 五层单向晋升 L1→L5 | 五层单向晋升 L1→L5 |
| 迭代策略 | 覆盖式更新，无时间衰减 | 时间衰减+重要度动态筛选 |
| 容量占比 | 20% | 80% |
| 子漏斗上限 | Nmax=8 | Nmax=12 |

系统采用**三维量化重要度公式**，作为所有记忆晋升、留存、收敛的核心判定依据：

$$I = I_0 + \alpha·S + \beta·V + \gamma·C$$

| 驱动信号 | 定义 | 应用场景 |
|----------|------|----------|
| S 安全显著性 | 事件的安全敏感等级 | 密码、资金、隐私类操作优先级拉高 |
| V 用户满意度 | 执行结果与用户期望契合度 | 用户确认、无修改行为提升记忆权重 |
| C 复用频次 | 策略成功复用的累计次数 | 高频有效策略沉淀为长期技能 |

依托**宏观自收敛机制**，子漏斗达到数量上限后自动归并合并。零散经验逐步抽象为高阶策略，让智能体行为从随机分散趋于稳定统一。

V1.1 核心时序保障：未完成会话、异常任务绝不晋升长效记忆。多会话分片隔离写入，全程保障记忆数据纯净无污染。

### 4.3 MCC 执行小脑（40模块 · 唯一落地出口）
MCC 为全系统唯一底层执行出口，纯工程状态机驱动，零智能决策能力，极致稳定可控。负责将 ECC 标准化意图，转化为跨平台原子操作。

核心能力包含双签名安全校验、DAG 任务依赖解析、并发限流管控、人机仿真防抖、数据脱敏、多窗口与网络资源管控、批量文件处理。内置分支独立快照撤销栈、七大类故障分级处理、整机资源动态降级能力，适配 Windows、macOS、Android、iOS 四端统一运行。

通过文件、剪贴板、窗口、网络四大全局互斥锁，彻底解决并发数据覆盖、资源抢占问题。搭配 WAL 预写日志、心跳卡死检测、断点续传机制，实现所有操作可审计、可回滚、可恢复、可追溯。

### 5.4 核心时序：会话只读锁机制
该机制为 V1.1 核心时序创新，彻底解决并发记忆污染、异步数据错乱问题，全系统统一强制执行：

1. ECC 发起 LLM 调用瞬间，下发当前会话记忆只读锁
2. 会话执行全程，任务数据仅缓存至临时层，禁止写入长效记忆
3. 单分支子任务完成不解锁、不沉淀、不归档
4. 仅全会话所有并行、串行任务正常执行完毕，方可解除锁限制
5. 系统低负载时段批量审核合规经验，完成长效记忆晋升归档

## 九、工业化落地与稳定性体系
EM-Core V1.1 是一套可量产、可长期维护、可商业化迭代的工业化 AI Agent 原生架构，稳定性与工程性远超传统脚本式智能体。

- **架构层面**：模块编号永久固定、架构冻结、能力子单元化，迭代不破坏底层根基。
- **运行层面**：总线隔离、资源锁隔离、故障分支隔离、动态降级保护、断点续传恢复，全程稳可控。
- **工程层面**：全链路可审计、可追溯、可热更新、可跨设备迁移、多平台统一适配。
- **部署层面**：端侧轻量化运行，无需强云端依赖，用户隐私本地可控，适配手机、PC全场景终端。
